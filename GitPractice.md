# Введение в контроль версий

## Git. Работа с терминалом

<br>

**Освновные команды**

<br>

- git init — *команда, создающая новый репозиторий в выбранной папке*

- git status — *посмотреть статус*

- git add < file name > — *добаление файла. Даже для отслеживаемых файлов необходимо вызывать команду git add.*

- git commit -m “comment” — *сохраняет изменения и подтверждает, что эти изменения работоспособны и не накладываются на правки других разработчиков. Оставлять комментарий к коммиту обязательно, чтобы было понятно, какие изменение были произведены*

###### Заметка: чтобы сделать commit, нужно обязательно сохраниться (ctrl + s), затем добавить файл в отслеживаемое (git add .\fileName)
- git log — *журнал изменеий, так же показывает номера последних сохранений*

- git checkout < save number > — *открывает версию под данным сохранением*

- git checkout main — *возвращение к мастер ветке*

- git commit -amend -m “…” или git commit --amend “…” — *редактировать название ПОСЛЕДНЕГО коммита*

- clear — *очищает видимую зону терминала*

- git clone — *это утилита командной сторки Git для выбора существующего репозитория и создания его клона, то есть копии*

- git branch — *позволяет создавать, просматривать, переименовывать и удалять ветки.*

###### Заметка: она не дает возможности переключаться между ветками или выполнять слияние разветвленной истории. Именно поэтому команда git branch тесно связана с командами git checkout и git merge
#### Кстати! Чтобы из репозитория снова сделать папку, достаточно всего лишь удалить скрытую папку «.git»
